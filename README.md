# Rebex — RL-Powered Medicine Stockout Prevention for Eritrean District Hospital Pharmacies

> BSc. Software Engineering Capstone Project — Rodas Goniche | Supervisor: Simeon Nsabiyumva

**Rebex** uses Deep Q-Network Reinforcement Learning to recommend medicine replenishment orders in district hospital pharmacies, reducing stockouts caused by unpredictable demand, seasonal surges, and supply chain disruptions.

[[Demo video](https://canva.link/blscdff2ud0uisp)]

---

## Background

Essential medicine availability in Eritrean district hospitals sits at around **80%** — roughly 1 in 5 prescribed medicines is unavailable at the point of care. Traditional methods like Economic Order Quantity (EOQ) and periodic manual reviews cannot adapt to the volatile demand patterns and supply constraints in low-resource settings.

Rebex trains a DQN agent in a simulated pharmacy environment, pairing it with a Hidden Markov Model (HMM) that classifies current demand into one of three regimes — *stable*, *surge*, or *disruption* — so recommendations adapt in real time.

---

## How It Works

1. A pharmacy technician logs daily stock counts via the web dashboard
2. The DQN agent analyses current inventory state and demand regime, then recommends an order quantity per medicine
3. The pharmacy manager reviews, adjusts if needed, and approves the order
4. Performance is tracked over time and compared against the EOQ baseline

---

## Features

- **RL-powered order recommendations** — per-medicine quantities generated by the trained DQN agent
- **Demand regime detection** — HMM identifies stable / surge / disruption states to contextualise recommendations
- **Stockout risk dashboard** — visual alerts for medicines approaching critical stock levels
- **Manager approval workflow** — no order is submitted without human review
- **Performance reports** — stockout frequency, service level, holding cost, and medicine waste trends
- **EOQ baseline comparison** — side-by-side metrics against the traditional method

## Getting Started

**Prerequisites:** Docker, Docker Compose, Git

```bash
git clone https://github.com/rodwol/rl_pharmaceutical_optimization.git
cd rl_pharmaceutical_optimization
docker-compose up --build
```

Access the dashboard at the URL shown in your terminal after startup.

### Train the RL agent

```bash
python rl/train.py --episodes 1000 --save-path models/dqn_agent.pt
```

### Development mode (without Docker)

```bash
pip install -r requirements.txt
docker-compose up db          # database only
uvicorn app.main:app --reload --port 8000
streamlit run 
```

## Evaluation

Performance is measured against an EOQ baseline across four metrics:

| Metric | Definition |
|---|---|
| Stockout Frequency | How often a medicine reaches zero stock |
| Service Level | Percentage of demand met from available stock |
| Holding Cost | Cost of maintaining excess inventory |
| Medicine Waste | Stock expired before use |

> Results will be updated as training progresses.

## Limitations & Future Work

**Limitations**
- No real transaction-level pharmacy data was available; the agent was trained on synthetic data calibrated from published literature
- Reward function tuning (balancing stockout avoidance, overstocking, and expiry costs) remains an open problem
- HMM belief state integration adds pipeline complexity

**Future work**
- Shadow-mode pilot at a district hospital pharmacy — log agent recommendations alongside staff decisions without disrupting operations
- Multi-medicine, multi-facility supply chain optimisation
- Live eLMIS data integration
- Comparison with PPO and A3C algorithms
